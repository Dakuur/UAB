Exercici 2: Preparaci de les dades (6 punts)

En aquest cas tractem amb un problema univariant, ja que noms hem seleccionat una de les variables o els atributs disponibles.

Per hem de seleccionar dos parmetres molt importants:

  • El nombre de dies previs que farem servir per al model predictiu.
  • El nombre de passos a qu volem formular les prediccions.

Per aquest cas, suposarem que volem fer una predicci a 15 dies. s a dir, volem que el model d'aprenentatge automtic ens mostri els valors futurs per als propers 15 dies. Aquest parmetre es defineix a partir dels requeriments del problema, per lgicament cal tenir en compte que les prediccions a llarg termini solen tenir un error molt ms gran que les prediccions a curt termini.

Pel que fa al nombre de dies previs que emprar el model per generar la predicci, aquest parmetre generalment involucra algunes proves per determinar el valor ptim o, almenys, un valor acceptable. En el nostre cas, desprs de fer algunes proves hem decidit fixar el nombre de dies previs en 15 dies, com en el cas anterior.

En resum, el model utilitzar els valors reals del volum d'aigua dels darrers 15 dies per generar una predicci del volum per als propers 15 dies.

Exercici 3. Conjunts d'entrenament i test (2 punts)

A l'aprenentatge supervisat, el conjunt d'entrenament i el conjunt de test sn dos conjunts de dades diferents utilitzats per entrenar i avaluar un model predictiu.

  • Conjunt d'entrenament: Aquest conjunt de dades es fa servir per entrenar el model. Consisteix en exemples d'entrada (variables independents) juntament amb les sortides corresponents (variables dependents o etiquetes). El model aprn aquestes dades per poder fer prediccions precises.
  • Conjunt de test: Aquest conjunt de dades s'utilitza per avaluar el rendiment del model un cop ha estat entrenat. Les dades de test (o prova) sn diferents de les dades d'entrenament i no es fan servir durant el procs d'entrenament. S'utilitzen per mesurar el model davant de dades noves i no vistes anteriorment. El rendiment del model s'avalua comparant les prediccions del model amb les etiquetes veritables en el conjunt de test.

Dividir el conjunt de dades en un conjunt d'entrenament i un conjunt de test s crucial per avaluar la capacitat del model per generalitzar dades noves i no vistes. Aix ajuda a evitar el sobreajustament del model al conjunt de dades d'entrenament i permet estimar com es comportar el model a la prctica.

Es demana implementar:

  1. Dividir el conjunt de dades (X i y) en dos subconjunts (disjunts) per entrenament i test. Assignarem un 30% de les dades a test, i el 70% restant a dades d'entrenament.

La sortida d'aquest procs han de ser quatre variables per a l'entrenament del model predictiu, amb els noms que s'indiquen a continuaci:

  • train_X: variable amb les dades d'entrada del model ($X_i$). En aquest cas concret, tindrem 15 valors per a cada fila, ats que hem determinat utilitzar els 15 dies anteriors per fer els pronstics.
  • test_X: variables amb les dades d'entrada del model per reservades per a la fase de test.

  • test_Y: variables amb les dades de sortida del model, per reservades per a la fase de test.

from sklearn.metrics import mean_squared_error
from math import sqrt


# funci per evaluar les prediccions vs reals
def evaluate_forecasts(actual, predicted):
    scores = list()
    # calculem RMSE per cada dia
    for i in range(actual.shape[1]):
        # calculem MSE
        mse = mean_squared_error(actual[:, i], predicted[:, i])
        # calculem RMSE
        rmse = sqrt(mse)
        # store
        scores.append(rmse)
    # calcul global de RMSE
    s = 0
    for row in range(actual.shape[0]):
        for col in range(actual.shape[1]):
            s += (actual[row, col] - predicted[row, col])**2
    score = sqrt(s / (actual.shape[0] * actual.shape[1]))
    return score, scores


# calculem l'error en les prediccions del conjunt de test
